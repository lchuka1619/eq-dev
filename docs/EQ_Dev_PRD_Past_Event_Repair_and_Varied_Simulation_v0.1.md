# EQ Dev — Past Event Repair & Varied Simulation PRD v0.1

**Огноо:** 2026-07-24  
**Сүүлийн шинэчлэлт:** 2026-07-24 — mastery model ба immersive vision  
**Статус:** Personal Practice pilot  
**Бүтээгдэхүүний ангилал:** Human Skills Simulation Coach

## 1. Асуудал

Олон хүнтэй газар, байгууллагын эвент, багийн ажиллагаа, тайзнаас зайлсхийдэг хэрэглэгч анхнаасаа “нийгэмд дургүй” байх албагүй. Олон жилийн жижиг ба том таагүй туршлага дараах мөчлөг үүсгэж болно:

`Таагүй мөчүүд хуримтлагдах → ичгүүр/түгшүүр нэмэгдэх → ижил орчныг аюултай гэж таамаглах → зайлсхийх → түр тайвшрах → зайлсхийлт батжих`

Одоогийн 7 өдрийн зам ба Arena нь ирээдүйн яриаг давтуулдаг боловч:

- өмнөх муу туршлагыг мөчүүдэд задлахгүй;
- нэг өдрийн “бүхэлдээ fail” гэсэн ерөнхий дүгнэлтийг шалгахгүй;
- нэг чадварыг өөр нөхцөлд шилжүүлэн ашиглах controlled variation байхгүй;
- зураг, дуу, POV клипийг аажмаар нэмэх media ladder байхгүй;
- simulation-аас бодит амьдрал руу шилжих optional bridge бүрэн тодорхойгүй.

## 2. Product insight

> Хэрэглэгч зөвхөн ирээдүйн эвентээс айдаггүй. Өмнөх ижил үйл явдлаас үлдсэн ичгүүр, бүтэлгүйтлийн ерөнхий дүгнэлт дахин мэдрэгдэхээс хамгаалж байж болно.

Апп онош тавихгүй, өнгөрсөн үйл явдлыг “төгс болгон дахин бичихгүй”. Харин:

1. тухайн туршлагыг жижиг мөчүүдэд задлана;
2. болсон баримт ба өөрийн тухай гаргасан дүгнэлтийг салгана;
3. нэг мөчид хэрэглэх өөр response/recovery-г аюулгүй давтуулна;
4. ижил чадварыг бага зэрэг өөр нөхцөлд олон давтуулна;
5. хүсвэл бодит амьдралын маш жижиг алхамд холбоно.

## 3. Pilot persona ба seed journey

**Persona:** Олон хүнтэй байгууллагын үйл ажиллагаанаас зайлсхийдэг, solo-first дасгал хүсдэг, шууд бодит уулзалт/тайзны даалгавар авбал аппыг орхих эрсдэлтэй хэрэглэгч.

**Seed history — нэг өдрийн ideation event:**

| Мөч | Туршлага | Болзошгүй ерөнхий дүгнэлт | Repair skill |
|---|---|---|---|
| Автобус | Хажуугийн хүнтэй чөлөөтэй байж чадаагүй | “Би хүнтэй эвтэйхэн байж чаддаггүй” | Minimal small talk + quiet permission |
| Багийн ажил | Уялдаа муу | “Би багт хэрэггүй” | Structure-giving role |
| Санаа давхцах | Урам хугарсан | “Миний санаа үнэ цэнгүй” | Differentiate or synthesize |
| Тайз | Үр дүн муу санагдсан | “Би олны өмнө бүтэлгүйтсэн” | One point + one example + close |
| Буцах үе | Бүхэл өдөр fail мэт болсон | “Дахин ийм эвентэд очиж болохгүй” | Evidence-balanced reflection |

**Дараагийн journey:** “Байгууллагын эвентэд тайван оролцох”.

## 4. Jobs to be done

- Өмнөх эвентийн яг аль мөч дараагийн эвентээс зайлсхийхэд нөлөөлж байгааг аюулгүй таних.
- “Юу болсон?” ба “Энэ миний тухай юу гэсэн үг гэж би дүгнэсэн?”-ийг салгах.
- Нэг хүнд мөчид ашиглах үг, завсарлага эсвэл recovery action-г давтах.
- Цээжилсэн ганц хариулт биш, бага өөрчлөлттэй нөхцөлд дасан зохицох.
- Бодит эвентэд “хамгийн нээлттэй хүн болох” биш, өөрийн хэмжээнд оролцох.

## 5. Амжилтын тодорхойлолт

Pilot journey-ийн амжилт:

- хэрэглэгч experience map-аас нэг мөч сонгосон;
- fact/conclusion ялгалтыг дуусгасан;
- нэг repair response-г хоёр controlled variant дээр давтсан;
- intensity нэмэгдвэл pause/level down/safe finish ашиглаж чадсан;
- session-ийн дараа “дараагийн удаа хийж болох хамгийн жижиг алхам”-аа сонгосон эсвэл penalty-гүй алгассан.

Бодит эвентийн амжилтын жишээ:

- автобусанд суух;
- хоёр хүнтэй мэндлэх;
- нэг хоолонд бусадтай суух;
- нэг тоглоом/ажилд жижиг үүрэг авах;
- явах хүсэл төрөхөд 5–10 минут үлдэх;
- завсарлага аваад буцаж нэгдэх.

Бүжиглэх, бүх хүнтэй ярилцах, тайзан дээр төгс байхыг success condition болгохгүй.

## 6. Core experience

### 6.1 Past Event Map

Хэрэглэгч:

- ижил өмнөх үйл явдал байгаа эсэхийг сонгоно;
- 3–7 мөчөөс өөрт хүчтэй нэгийг сонгоно;
- тухайн үеийн intensity-г optional тэмдэглэнэ;
- дэлгэрэнгүй бичихгүйгээр preset ашиглаж болно;
- хүссэн үед алгасаж шууд future rehearsal руу орж болно.

### 6.2 Fact vs Conclusion

Хоёр богино талбар:

- **Болсон зүйл:** ажиглагдахуйц үйл явдал.
- **Надад үлдсэн дүгнэлт:** өөрийн тухай эсвэл ирээдүйн тухай тайлбар.

AI нь оношлохгүй, маргахгүй. Нэг balanced reframe санал болгоно. Жишээ:

> “Багийн ажиллагаа муу байсан” нь баримт байж болно. “Тэгэхээр би багт хэрэггүй” нь тухайн өдрийн дараа үлдсэн дүгнэлт байж болно.

### 6.3 Repair Rehearsal

Өнгөрснийг өөрөөр болсон мэт дүрслэхгүй. Сонгосон мөчид:

- нэг response;
- нэг recovery phrase;
- нэг жижиг action

гэсэн сонголтоос нэгийг давтуулна.

### 6.4 Controlled Variation Engine

Scene нь дараах slot-оос эвлэнэ:

| Slot | Жишээ |
|---|---|
| Environment | автобус, хоолны хэсэг, багийн ширээ, тайзны өмнө |
| Persona | танил хүн, дуу цөөнтэй хүн, наргианч хүн, ахлах ажилтан |
| Opening | мэндчилгээ, асуулт, чимээгүй байдал, хөнгөн шүүмжлэл |
| Friction | суудал дүүрсэн, санаа давхцсан, дүрэм ойлгомжгүй |
| Goal | мэндлэх, асуух, бүтэц санал болгох, 5 минут үлдэх |
| Difficulty | guided, prompted, light surprise |
| Media | зураг, optional ambient audio, optional POV clip |

Дүрэм:

- session бүр нэг `target_skill`-тэй;
- AI зөвхөн зөвшөөрөгдсөн slot утгыг сонгоно;
- difficulty нэг session дотор нэг шатнаас илүү өсөхгүй;
- light surprise нь доромжлол, айлган сүрдүүлэлт, хүчтэй дайралт болохгүй;
- өмнөх variant-тай яг ижил opening + persona + friction хослолыг шууд давтахгүй;
- хэрэглэгч replay хүсвэл deterministic seed ашиглана.

#### Mastery ба progression policy

Нэг чадварыг “нэг удаа зөв хэлсэн”-ээр эзэмшсэн гэж тооцохгүй:

`introduced → repeating → stabilizing → independent → adapting → consolidating`

Progression decision нь `repeat | soften | progress | consolidate | pause` байна. Шийдвэрийг сүүлийн 3 variant-ийн outcome, prompt/hint/recovery usage, before/after intensity, user confidence/difficulty, safe finish/abrupt quit, өөр өдөрт дахин хийсэн эсэхийг хамтатган гаргана.

Нэг self-rating, нэг LLM score эсвэл нийт completion count дангаараа difficulty өсгөхгүй. Эхний content heuristic нь **70% familiar / 20% small variation / 10% light surprise**; нэг давталтаар 1–2 slot л солигдоно. Шат ахисны дараа consolidation variant өгнө.

### 6.5 Media ladder

1. Static image
2. Image + ambient audio
3. 5–15 секундийн POV clip
4. Guided response
5. Prompted response
6. Light surprise
7. Connected 6–8 decision rehearsal

Autoplay үгүй. Mute, text-only, skip, level-down үргэлж боломжтой. Pilot эхний release-д static image + optional ambient audio хангалттай; video нь дараагийн content-pipeline validation.

### 6.6 Real-life bridge

Bridge нь:

- optional;
- нэг л жижиг action;
- no streak penalty;
- intensity 8–10 үед default-аар нуусан;
- “хийсэн/хийгээгүй” бус “юу ажигласан?” reflection-тэй байна.

### 6.7 Entry router

Today дээр бүх feature-ийг зэрэг сонгуулахгүй. Нэг богино readiness check-ээр:

1. `past_repair` — өмнөх ижил үйл явдлын дурсамж саад болж байгаа;
2. `future_rehearsal` — ойрын тодорхой эвентэд бэлтгэх;
3. `daily_skill_loop` — тодорхой эвентгүйгээр нэг чадвар тогтворжуулах

гэсэн замын нэгийг санал болгоно. Recommendation тайлбартай, override боломжтой, Past Event Repair үргэлж optional байна.

### 6.8 Long-term immersive vision — Social Flight Simulator

> Хэрэглэгч бодит нийгмийн нөхцөлд орохоос өмнө, ижил шийдвэрлэх мөчүүдийг аюулгүй виртуал орчинд хангалттай олон хувилбараар давтаж, бие даан дасан зохицож сурна.

| Шат | Experience | Батлах зүйл |
|---|---|---|
| 1 | Text + voice + static image | Learning loop ба safety |
| 2 | Ambient audio + branching POV clip | Immersion нэмэгдэхэд completion хадгалагдах эсэх |
| 3 | Mobile 360° event recording | Presence нь real-life transfer-д нөлөөлөх эсэх |
| 4 | VR headset interactive rehearsal | Харц, зай, бүлэг, тайзны дарамтыг graded байдлаар дасгах |
| 5 | AI-generated interactive event world | Нэг skill дээр төгсгөлгүй controlled variants |

Бүх шат нэг canonical contract ашиглана:

`Scene → target skill → decision moments → characters → user response → reaction → feedback → variation`

Event recording нь зөвшөөрөлтэй жүжигчид/оролцогч ашиглах, 6–8 интерактив decision moment болгон annotation хийх, нүүр/дуу/байгууллагын мэдээллийн usage scope ба retention-г бүртгэх, consent цуцлагдвал asset-ийг unpublish/delete хийх боломжтой байх шаардлагатай. Passive replay нь practice completion биш.

## 8. Data model

```ts
type PastEvent = {
  id: string;
  userId: string;
  journeyId: string;
  title?: string;
  eventType: "work_event" | "group_activity" | "presentation" | "other";
  createdAt: string;
};

type PastEventMoment = {
  id: string;
  pastEventId: string;
  momentType: string;
  factText?: string;
  conclusionText?: string;
  intensity?: number;
  selectedForRepair: boolean;
};

type SceneTemplate = {
  id: string;
  targetSkill: string;
  allowedEnvironments: string[];
  allowedPersonas: string[];
  allowedOpenings: string[];
  allowedFrictions: string[];
  allowedGoals: string[];
  difficultyLevels: Array<"guided" | "prompted" | "light_surprise">;
  mediaAssetIds: string[];
  safetyTags: string[];
};

type SceneVariant = {
  id: string;
  templateId: string;
  seed: string;
  environment: string;
  persona: string;
  opening: string;
  friction: string;
  goal: string;
  difficulty: "guided" | "prompted" | "light_surprise";
};

type SkillMastery = {
  userId: string;
  targetSkillId: string;
  stage:
    | "introduced"
    | "repeating"
    | "stabilizing"
    | "independent"
    | "adapting"
    | "consolidating";
  recentVariantIds: string[];
  independentSuccessCount: number;
  hintUseCount: number;
  safeFinishCount: number;
  lastPracticedAt: string;
};

type DecisionMoment = {
  id: string;
  sceneId: string;
  targetSkillId: string;
  cueStartMs?: number;
  cueEndMs?: number;
  allowedResponseModes: ("voice" | "text" | "choice" | "gaze" | "gesture")[];
  branches: string[];
};

type MediaAsset = {
  id: string;
  renderer: "text_voice" | "image_audio" | "pov_video" | "video_360" | "vr_interactive";
  locale: string;
  consentRecordId?: string;
  usageRights: string;
  retentionPolicy: string;
  decisionMomentIds: string[];
};

type RealLifeBridge = {
  id: string;
  sessionId: string;
  actionKey: string;
  status: "offered" | "accepted" | "skipped" | "reflected";
  beforeIntensity?: number;
  afterIntensity?: number;
  observation?: string;
};
```

`factText`, `conclusionText`, reflection, transcript-ийг sensitive personal data гэж үзнэ. Raw audio default-аар хадгалахгүй. Manager/organization эдгээрийг харахгүй.

## 9. AI contracts

AI response structured JSON байна:

```json
{
  "balanced_reframe": "string",
  "repair_options": [
    {"type": "response", "text": "string"},
    {"type": "recovery", "text": "string"},
    {"type": "action", "text": "string"}
  ],
  "safety": {
    "requires_level_down": false,
    "avoid_surprise": false
  }
}
```

Validation fail бол curated local fallback ашиглана. AI:

- trauma/mental-health diagnosis тавихгүй;
- хэрэглэгчийн дурсамж үнэн/худал гэж шийдэхгүй;
- “та заавал оч” гэж шахахгүй;
- нэг удаад нэгээс олон improvement өгөхгүй;
- өндөр intensity үед surprise үүсгэхгүй.

## 10. Analytics

Primary:

- `past_event_repair_completion_rate`
- `first_to_second_variant_completion_rate`
- `pre_post_intensity_change`
- `safe_finish_without_abrupt_quit_rate`
- `bridge_acceptance_rate`
- `day_2_return`, `day_7_completion`

Guardrail:

- experience check дээр abrupt quit
- media эхэлсний дараах abrupt quit
- intensity +3 буюу түүнээс их өссөн session
- repeated low-confidence STT
- bridge screen abandonment

Qualitative:

- “Энэ дасгал өмнөх туршлагыг ойлгоход тусалсан уу?”
- “Хоёр дахь хувилбар хэрэгтэй өөрчлөлт байсан уу?”
- “Дараагийн бодит алхам боломжтой мэт санагдсан уу?”

## 11. Out of scope

- PTSD/trauma diagnosis эсвэл treatment claim
- EMDR, exposure therapy-г эмчилгээ хэлбэрээр автоматжуулах
- хэрэглэгчийг олон хүнтэй эвентэд заавал явуулах
- олон past event-ийг нэг session-д гүн боловсруулах
- raw audio-г default хадгалах
- manager-д individual response/reflection харуулах
- open-ended AI-generated surprise
- Pilot release дэх VR/avatar video implementation
- эхний pilot-д user-generated video upload

360°/VR нь pilot implementation scope-д орохгүй боловч data/content contract болон media renderer boundary-д урт хугацааны requirement хэлбэрээр орно.

## 12. Pilot acceptance criteria

- Нэг ideation seed event-ийг 5 moment болгон харуулна.
- Хэрэглэгч нэг moment сонгож fact/conclusion-г edit эсвэл skip хийж чадна.
- Нэг repair option сонгож text/voice-аар давтана.
- Нэг target skill дээр дор хаяж 3 deterministic variant байна.
- Static image ба optional ambient audio ажиллана; autoplay үгүй.
- Pause/hint/level-down/safe-finish нэг товшилтын зайд байна.
- Intensity 8–10 үед surprise болон bridge default-аар нуугдана.
- Refresh/network алдаанд session state сэргээнэ.
- Raw audio хадгалахгүй; sensitive fields manager analytics-д гарахгүй.
- Analytics нь repair, variation, safety, bridge funnel-ийг хэмжинэ.
- Mastery progression нь нэг self-rating/LLM score-д тулгуурлахгүй; repeat/soften/progress/consolidate/pause decision тесттэй байна.
- Нэг target skill 2–3 controlled variant болон өөр өдөрт дор хаяж нэг дахин баталгаажсаны дараа independent/adapting шат руу орно.
- Scene contract нь renderer-agnostic байна; text/voice logic-ийг ирээдүйн 360°/VR-д зориулан дахин бичих шаардлагагүй.
